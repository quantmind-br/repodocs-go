# Security Policy

## Supported Versions

RepoDocs appears to be in pre-1.0 development. Security fixes are applied to the default branch unless maintainers publish a supported release policy.

| Version | Supported |
|---------|-----------|
| `main`  | Yes       |
| Older commits or forks | No |

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Report suspected vulnerabilities privately by contacting the maintainers through the repository owner's preferred GitHub contact method. If a private security advisory option is available on GitHub, use it.

Include as much detail as possible:

- Affected component or command.
- Steps to reproduce.
- Potential impact.
- Suggested fix or mitigation, if known.

## Disclosure Policy

Maintainers will acknowledge valid reports as soon as practical, investigate the issue, and coordinate a fix before public disclosure. Public disclosure should happen only after a patch or mitigation is available, unless maintainers and reporter agree otherwise.

## Contact Method

Use GitHub private vulnerability reporting or the repository owner's public GitHub contact method. If neither is available, open a minimal public issue asking for a private security contact without disclosing vulnerability details.
